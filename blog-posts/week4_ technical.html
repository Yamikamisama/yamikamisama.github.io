The Map Method: How it Works

Map is one of the most useful methods that can be used on Arrays; most people think that it belongs to the Array class, but sneaky map is actually a method of the Enumerable module.

What does map look like?

How does map work?

When you are mapping an array you iterate through the array and pass each element to a block that does what you ask and creates a new array. Here is an example of some code to show how it works.

arr1 = ["this", "is", "array"]
=> ["this", "is", "array"]

***Here we are simply making an array***

arr1.map { |x| x * 2 }
=> ["thisthis", "isis", "arrayarray"]

***Map is iterating through each element in the array and multiplying it by two***

arr1
["this", "is", "array"]

***This is checking if the array has been changed permanently (which it hasn’t), lets see how we do that***

arr1.map!{|x| x * 2 }
["thisthis", "isis", "arrayarray"]

***To make permanent changes use the ! which makes the method destructive. BOOM out with the old and in with the new***

arr1
["thisthis", "isis", "arrayarray"]

***Checking one more time to see that it made the changes permanent. I always like to double check that Ruby is doing what I’m telling it to do! ☺***
